# JavaScript에 관하여

## JavaScript란 무엇인가?

1. JavaScript 는 런타임에 컴파일되는 동적 약형 프로그래밍 언어입니다. 이는 브라우저에서 웹페이지의 일부로 실행될 수도 있고 소위 호스트 환경이라 불리는 모든 머신에서 직접 실행될 수도 있습니다.

2. JavaScript 는 웹페이지를 보다 동적으로 만들기 위해 생성되었습니다. 이게 바로 JavaScript가 과거에 탄생한 이유라고 볼 수 있다. 브라우저 내에서 페이지의 내용을 직접 변경하기 위해 만들어졌고 최초엔 LiveScript라고 불렸지만 당시 Java의 인기로 인해 Java와 유사성을 꾀하여 JavaScript 로 이름이 변경 되었다. - 그러나 Java와 JavaScript는 완전히 별개로 공통점이 없다

## JavaScript는 어떻게 실행될까?

JavaScript 코드를 실행하려는 모든 환경에 내장된 중요한 한 가지가 있는데 그게 바로 JavaScript 엔진이다. 브라우저 내에 내장되어 있는데 엔진의 역할은 코드를 구문 분석하고 머신 코드가 보다 빨리 실행되기 때문에 머신 코드로 즉성 컴파일링을 합니다. 모든 과정이 이루어지고 코드가 실행되면 웹페이지에 효과가 나타난다.

오늘 날의 컴퓨터는 동시에 여러 작업을 수행할 수 있는 다중 스레드가 지원되지만 JavaScript의 코드 실행은 하나의 단일 스레드에서 실행이 됩니다.

## 동적 vs 약형 언어

JavaScript는 동적 해석형 언어이면서 약형 프로그래밍 언어이기도 합니다.

#### 동적 해석형 언어란?

> 미리 컴파일 되지 않았음을 의미합니다.  
> JavaScript는 전송 중 컴파일링 되기 때문에 코드가 런타임에 평가되고 실행 된다. 이는 또한 코드가 런타임에서 변경될 수 있죠. 코드 내에서 다른 프로그래밍 언어에서는 허용되지 않는 일부 작업이 이루어질 수 있다.

#### 약형 언어란?

> JavaScript는 텍스트, 숫자와 같은 데이터의 정의를 내리지 않아도 된다. 어떤 데이터를 저장하든 그대로 받아 들인다. 따라서 데이터 유형은 고정되어 있지 않으며 바꿀 수가 있다.

## 3.2 var, let, const

1. var

- let 과 마찬가지로 변수를 생성합니다.
- var 는 자바스크립트가 나오면서 부터 변수를 생성하기 위해 사용 하였다.
- var 는 함수, 전역 스코프에서 변수 생성이 가능하다.

### Hoisting

> 호이스팅(Hoisting) 이란?
> 함수 안에 있는 선언들을 모두 끌어올려서 해당 함수 유효 범위의 최상단에 선언하는 것을 의미하며 실제 코드가 끌어 올려지는 것은 아니며, 자바스크립트 Parser 내부적으로 끌어 올려져서 처리 하는것이다.
> var변수/함수 선언과 함수 선언문만이 호이스팅이 일어난다.

2. let

- var 와 마찬가지로 변수를 생성합니다.
- let 은 ES6 버전부터 사용 가능해졌다.
- let 은 블록 스코프에서 변수 생성이 가능하다.

3. const

- const 는 단독적으로 상수를 생성합니다.
- const 는 ES6 버전부터 사용 가능해졌다.
- const 는 블록 스코프에서 상수 생성이 가능하다.

## 3.3 코드는 어떻게 실행 되는가

- 자바스크립트 엔진 - 코드를 실행하는 주체이며 여기서 메모리와 실행 단계에 대한 관리가 이루어 진다.  
   여기에는 중요한 개념이 두가지 있는데 그것이 힙(Heap)과 스택(Stack)이다.  
   3.자바스크립트 - 1. 힙,스택 - app.js 주석에 세부내용 추가

  #### 힙(Heap)

  > 힙은 장기 메모리를 뜻하며 시스템 메모리 데이터를 저장한다.
  > 브라우저가 힙을 관리하고 시스템 메모리에 대한 작업을 수행

  #### 스택(Stack)

  > 스택은 단기 메모리와 관련된 작업을 수행
  > 코드를 실행할 때 중요한 역할을 하는데 프로그램 흐름을 관리, 현재 실행되고 있는 함수를 관리하는 역할을 한다.

  ### 원시 Vs 참조 값

  - 원시값 :

  > 데이터의 6개 자료형이 원시 값에 속한다. (Strings, Numbers, Booleans, null, undefined, Symbol)
  > 메모리에 저장되고 보통 스택(Stack)이다.
  > 변수를 복사하면 값이 복사가 된다.

  - 참조값 :

  > 다른 모든 객체
  > 메모리(힙)에 저장되고 변수는 메모리의 위치에 대한 포인터를 저장합니다.
  > 변수를 복사하면 포인터나 참조값이 복사됩니다.

  #### 가비지 콜렉션(Garbage Collection)

  - 역할

  > 사용되지 않은 객체(참조되지 않은 객체)에 대한 힙 메모리를 주기적으로 확인
  > 사용되지 않은 객체를 메모리에서 제거합니다

  - 메모리 누수(Memory Leaks)

  > 코드를 작성할 때 더는 동작 하지않는 객체가 있고 여전히 이를 참조하고 있다면 이것에 관심이 있다면 가비지 컬렉션을 진행할 수 없다. 코드에서 확실히 원하지 않는다고 해두지 않았기 때문에 버그나 에러가 발생 할 수도 있다.

# 4. 함수

## 함수는?

> 온-디맨드(On-Demand) 형식으로 코드를 정의하는 구조이며 자바스크립트가 스크립트 내에 전체 내용을 보고 추가한 함수 선언을 찾아서 함수 등록을 한다.
> 함수 내에 상수나 변수를 생성하면 해당 함수에 속하게 되며 이는 블록 스코프 때문이다.
> 함수는 인수라는 매개변수를 취해 값을 반환하는데 함수는 아무것도 하지않고 함수가 필요한 경우에 매개변수를 취할 수 있다. 하지만 매개변수를 취하지 않고 아무것도 반환하지 않는 참수도 작성 가능
> 함수는 여러번 호출이 가능하며 다양하게 인수를 받을 수 있으므로 동적이다
> 함수는 직접적, 간접적으로 호출이 가능하다.
> 함수는 객체이다

매개변수 : 함수를 정의할 때 괄호 안에 지정하는 변수  
인수 : 함수를 호출할 때 함수에 전달하는 구체적인 값

#### 함수 선언 Vs 함수 표현식

> 함수 선언(함수 문장 구문)  
> Ex)

```
function multiply(a,b) {
   return a * b;
}
```

- 자바스크립트가 자동으로 함수를 맨위로 호이스트(Hoist)하고 초기화한다.

> 함수 표현식  
> Ex)

```
const multiply = function(a,b) {
   return a * b;
}
```

- 호이스팅(Hoisting) 자체도 작동하지만 상수가 정의되지 않은 상태로 호이스트(Hoist) 된다. 초기화가 이루어지지 않으니 정의 되기 전까지는 함수를 호출할 수가 없다.

#### Arrow function

- function 문법을 왜 사용할까?

1. 여러가지 기능을 하는 코드를 한 단어로 묶고 싶을 때(재사용 용이)
2. 입출력 기능을 만들 때[pp]

- Arrow function의 장점
  1. Arrow function 을 사용 하게 되면 더욱 쉽고 이쁘게 입출력 기능을 만들 수 있다.
  2. 소괄호, 중괄호, return 생략이 가능하다.
  3. 내부에서 this 값을 쓸 때 외부의 this 값을 그대로 사용 가능(단점이 될 수도 있다.)

#### Rest 연산자 (매개변수)

- 많은 인수로 함수를 호출해야 할 때 사용하기 좋다.
- 이 함수가 얻는 모든 인수를 많이 가져와서 배열에 합치는 것 - 함수 안에 배열을 설계하는 것
- 리스트의 마지막 인수여야 한다.
- 1개의 Rest 매개변수만 가질 수 있다. (자동으로 모든 인수를 합치기 때문)

#### 콜백함수

- 파라미터로 함수를 전달하는 함수이다.
- 변수의 유효범위나 동기/비동기 처리에 유용하다
- 코드의 재활용이나 관리가 쉬워진다

함수에 대한 것들을 간단하게 알아봤는데 함수는 사용하는 경우가 많고 더욱 알아갈 것들이 많기 때문에 더욱 연습과 공부가 필요

# 5. Dom 작업

## Dom 이란?

- Dom 은 'Document Object Model' 의 약자로 로드 및 렌더링 된 HTML 코드를 의미하며 자바스크립트와 브라우저를 통해 코드를 나타내주는 것이나 다름없다.

#### document vs window

1. Document

   - 브라우저가 노출시키는 루트 DOM 노드이다.

   - HTML 요소를 쿼리하거나 DOM 콘텐츠와 상호작용을 할 때 매우 유용하고 다양한 메서드와 기능을 제공한다.

2. window

   - document를 프로퍼티로 가지고 있다.

   - window 는 자바스크립트의 브라우저에서 사용 가능한 최상위 전역 객체이고 활성화된 브라우저 창이나 탭을 보여준다.

   - 스크립트의 전역 저장소 역할을 하며 윈도우 별 프로퍼티 및 메서드에 대한 액세스도 제공합니다.
